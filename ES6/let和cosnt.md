### var

1.变量声明提升

2.全局变量会绑定到window上

3.变量可以重复声明



### let

1.变量声明不提升

2.全局变量不会绑定到window上

3.变量不能重复声明

4.块级作用域

5.会产生临时死区(暂时性死区)

```js
由于let和const无法进行变量声明,因此在声明前访问变量会导致错误,这就是临时死区
let a = 1;
if(a >= 1){
       console.log(a)
    	let a = 5;
   }
在if括号中会有个临时作用域,若不在里面let a变量则控制台输出全局的变量a,但是由于if作用域中变量声明前访问量a变量,并且定义了let a,无法继续向上层作用域访问a,所以报错
```





### const

1.**一旦声明必须赋值**,不能使用null占位。

2.声明后不能修改　

3.如果声明的是复合类型数据，可以修改其属性(比如说对象)
